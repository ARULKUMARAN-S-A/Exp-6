# Exp - 6 STUDENT ONLINE EXAM REGISTRATION SYSTEM

# AIM:
To develop the Exam Registration System using Star UML tools.
# SRS (Procedure):
1. **Problem Identification** – Identify the need to conduct exams online efficiently and securely.
2. **Requirement Gathering** – Collect requirements from students, teachers, and administrators.
3. **System Analysis** – Study existing manual or offline exam methods and identify issues like time, errors, and security.
4. **System Design** – Design modules for question management, exam scheduling, student login, and result processing.
5. **Functional Specification** – Define main functions like add questions, conduct exams, auto-evaluate answers, and generate results.
6. **Non-Functional Specification** – Specify requirements for performance, security, reliability, and user-friendliness.
7. **Testing & Validation** – Test all modules to ensure correct question display, time limits, and result accuracy.
8. **Documentation & Maintenance** – Prepare reports, user guides, and update the system as per new exam needs.
